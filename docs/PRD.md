# 마이링크 (MyLink) - 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트 명**: 마이링크 (MyLink)
- **목적**: 개발자, 크리에이터 등 다수의 링크(포트폴리오, GitHub, 블로그, 소셜 미디어 등)를 보유한 사용자가 하나의 마이크로 랜딩 페이지로 링크를 통합하여 쉽게 공유할 수 있도록 지원하는 웹 서비스.
- **대상 사용자**: 프로그래머, 디자이너, 크리에이터 및 인플루언서 (본인의 다양한 온라인 활동을 한 곳에 모아 홍보하고 싶은 사용자)
- **디자인 컨셉**: `shadcn/ui` 기반의 모던하고 깔끔한 UI 구성 (별도의 테마 시스템은 완전히 배제)

## 2. 유저 플로우 및 핵심 경험
1. **간편한 시작**: 타겟 사용자인 개발자/크리에이터에게 익숙한 **구글 소셜 로그인**으로 단 1초 만에 가입 및 로그인.
2. **나만의 공간**: 가입 직후 `기본도메인/[username]` (예: 기본도메인/minwoo) 형태의 고유하고 직관적인 URL 할당.
3. **직관적인 관리**: 별도 페이지 이동이나 복잡한 모달 없이 **인라인 편집(Inline Edit)** 방식으로 즉시 내 정보와 링크를 수정. 

## 3. 기능 목록 (Features)

### 3.1. MVP (필수 핵심 기능)
| 카테고리 | 기능명 | 상세 설명 |
|---|---|---|
| **인증(Auth)** | 구글 소셜 로그인 | Firebase Auth를 활용한 구글 계정 연동 가입 및 로그인 단일 지원 |
| **프로필 관리** | 고유 URL 할당 | `기본도메인/[username]` 형태로 사용자별 고유 접속 페이지 제공 |
| | 인라인 정보 편집 | 활동용 닉네임(`displayName`), **실제 사용자 이름(Name)**, 한 줄 소개(Bio)를 인라인 편집으로 수정 (이미지 업로드 기능 제외) <br/> **(참고: `displayName`의 초기값은 가입 시 구글 Gmail 계정 이메일 아이디의 앞부분을 자동으로 추출하여 설정됨)** |
| **링크 관리** | 인라인 CRUD | 새로운 링크 등록(URL, 제목), 기존 링크는 **인라인 편집** 방식으로 정보 텍스트 즉각 수정 및 삭제 |
| | 구글 파비콘 API 연동 | 링크 렌더링 시 **구글 파비콘 API**(`https://www.google.com/s2/favicons?domain=...`)를 활용하여 등록된 사이트의 아이콘을 자동으로 렌더링 |
| **퍼블릭 뷰** | 반응형 페이지 | 외부 사용자가 접속할 때 다양한 기기에 맞는 UI 제공 (대시보드 내 관리자용 모바일 뷰 미리보기 화면은 제외) |

*(참고: 프로필 이미지 업로드 기능, 커스텀 테마 설정, 드래그 앤 드롭 순서 변경, 링크 활성화/비활성화 토글 기능은 제외된 컴팩트한 MVP입니다.)*

### 3.2. V2 (선택 및 추후 구현 기능)
| 카테고리 | 기능명 | 상세 설명 |
|---|---|---|
| **통계(Analytics)** | 링크별 클릭 조회수 | 개별 링크별로 몇 번의 클릭이 발생했는지 트래킹 및 조회수 제공 |

## 4. 기술 스택 및 데이터베이스 모델링
- **프론트엔드 UI**: Next.js (App Router), React, `shadcn/ui`
- **인증 인프라(BaaS)**: `Firebase Auth` (Google OAuth 단일 지원)
- **데이터베이스(DB)**: `Firebase Firestore`
  - **데이터 모델링 원칙**:
    - 사용자(User) 문서 내에 고유 식별자인 `username`(URL에 사용), 서비스 노출용 닉네임인 `displayName`(최초 구글 메일 아이디 앞부분 파싱 맵핑), 그리고 **실제 이름(이름, Name)** 필드를 모두 포함하며 수정 가능하도록 구성.
    - 각 사용자가 보유한 **링크 데이터는 서브컬렉션(Sub-collection)을 활용하여 분리 및 관리**.
