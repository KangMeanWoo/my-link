# 마이링크 (MyLink) - 사용자 시나리오 (User Scenario)

본 문서는 마이링크 서비스의 주요 액터(방문자와 소유자)가 서비스를 이용하는 핵심 과정을 행동 중심("**사용자는 [목적]하기 위해 [행동]한다**" 형태)으로 서술한 시나리오입니다.

---

## 1. 방문자 (Visitor) 시나리오

### 1.1 접속 및 프로필 확인
- 방문자는 **소유자의 통합된 링크 정보를 확인하기 위해** 브라우저 주소창에 URL(`기본도메인/username`)을 입력하거나 외부 SNS 프로필에 기재된 마이링크 주소를 클릭한다.
- 방문자는 **현재 보고 있는 페이지가 누구의 포트폴리오인지 정확히 알기 위해** 최상단에 노출된 소유자의 닉네임(`displayName`), 실제 이름, 그리고 한 줄 소개(Bio)를 확인한다.

### 1.2 링크 탐색 및 이동
- 방문자는 **소유자가 공유한 다양한 활동(깃허브, 블로그 등)을 훑어보기 위해** 화면에 스크롤 형식으로 깔끔하게 나열된 링크 리스트들을 탐색한다.
- 방문자는 **어떤 사이트로 연결되는지 직관적으로 파악하기 위해** 링크 항목마다 매핑되어 있는 아이콘(구글 파비콘 API 결과물)과 링크 제목을 함께 확인한다.
- 방문자는 **특정 포트폴리오나 소셜 미디어로 직접 접근하기 위해** 원하는 링크 영역을 마우스로 클릭(또는 터치)하여 해당 외부 사이트로 이동한다.

---

## 2. 소유자 (Owner) 시나리오

### 2.1 가입 및 본인 공간 할당
- 소유자는 **빠르게 마이링크 계정을 생성하고 접속하기 위해** 별도의 입력 항목 없는 구글 소셜 로그인 버튼을 클릭한다.
- 소유자는 **자신만의 고유한 링크 공유 주소를 획득하기 위해** 가입 직후 자동 생성된 `기본도메인/username` 형태의 사용자 고유 URL 페이지를 부여받는다.

### 2.2 프로필 조회 및 인라인 수정 (Update)
- 소유자는 **프로필에 자신의 구글 메일 아이디 앞부분이 닉네임으로 자동 부여된 것을 활동명으로 변경하기 위해** 해당 닉네임 영역 텍스트를 클릭하여 '인라인 편집(Inline Edit)' 모드를 활성화한다.
- 소유자는 **방문자에게 본명을 기반으로 더 높은 신뢰를 주기 위해** 실제 이름(Name) 영역과 한 줄 소개(Bio) 영역을 클릭하여 텍스트를 인라인으로 수정한 뒤 엔터(Enter) 키나 외부 여백을 클릭해 즉시 저장시킨다.

### 2.3 링크 추가 (Create)
- 소유자는 **사람들에게 새로 작성한 기술 블로그 글을 공유하기 위해** '링크 추가' 영역에서 목적지 URL 주소와 사람들이 알아보기 쉬운 제목을 입력한 뒤 등록 버튼을 클릭한다.
- 소유자는 **내가 올린 웹사이트의 로고가 잘 적용됐는지 확인하기 위해** 입력 직후 구글 파비콘 API를 통해 해당 블로그 아이콘이 링크 좌측에 자동으로 렌더링되는 작동을 확인한다.

### 2.4 기존 링크 수정 (Update)
- 소유자는 **기존에 올려둔 포트폴리오 주소가 변경되었거나 제목 오타를 고치기 위해** 등록된 링크의 제목 혹은 URL 텍스트를 직접 클릭하여 즉시 인라인 편집기를 연다.
- 소유자는 **수정 프로세스를 불편한 모달 창 이동 없이 처리하기 위해** 알맞게 타이핑 후 텍스트필드 바깥을 클릭하여 수정본을 안전하게 실시간 저장한다.

### 2.5 기존 링크 삭제 (Delete)
- 소유자는 **기간이 만료된 이벤트 링크를 방문자에게 노출되지 않게 하기 위해** (숨기기 토글 기능이 없으므로) 해당 링크 우측에 위치한 '삭제(휴지통)' 아이콘을 클릭한다.
- 소유자는 **실수로 중요한 링크를 날리는 것을 방지하기 위해** 화면에 나타난 '정말 삭제하시겠습니까?'라는 확인 모달창에서 최종적으로 '삭제 허용'을 누른다.
